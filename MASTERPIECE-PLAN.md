## 3. מה החוויה נותנת היום ומה חסר (מבחן-המשתמש)

**מה שעובד (על פרוד, במצב תבנית):** קטלוג קורסים ציבורי, 75 שיעורים עם תוכן, יועץ-תבניות שמזהה כוונה (דחייה/תקוע/איך/סיכום) ועונה בקול-אלעד עם הקשר-שלב, 4 דיאלוגים מובנים איכותיים (בחירות מנוקדות + טיפ פדגוגי לכל צעד), 6 פרסונות צ'אט, ציון+משוב בסוף.

**מה שחסר כדי להיות "הרמה הגבוהה ביותר":**
1. **היועץ לא מכיר את התוכן.** הוא יודע באיזה שלב אתה — אבל לא קרא אף שיעור, לא את הספר, לא את הטיפולוגיה. שאלה כמו "מה שלוש רמות המשיכה?" תקבל תשובה גנרית במקום ציטוט מדויק משבוע 6. (הפתרון: RAG — סעיף 5.)
2. **היועץ לא זוכר אותך.** כל ביקור בשיעור = שיחה מאפס. אין פרופיל-לומד, אין רצף.
3. **הפרסונות שטוחות.** "מיה, מעצבת גרפית, זהירה" — אין טיפוס מהטיפולוגיה של אלעד, אין מצב-רגשי שמתפתח, אין טריגרים, אין "מה מקרב/מה מרחיק".
4. **אין קשת רגשית.** הפרסונה באותו מצב-רוח בהודעה 1 ובהודעה 15. אין מתח, אין רגעי-מבחן, אין "היא נפתחת אליך כי עשית משהו נכון".
5. **המשוב שטחי.** ציון + 3 חוזקות + 3 שיפורים גנריים. אין ניתוח רגעים ("ברגע שהיא שיתפה על העבודה — פספסת"), אין קישור לשיעור הרלוונטי, אין "משחק חוזר".
6. **אין מדדי-התקדמות בין סימולציות.** כל תרגול הוא אי בודד.
7. **כיסוי תרחישים דל:** 4 דיאלוגים + 6 צ'אטים על 5 שלבי-קורס. שלבים 4-5 (חיבור, מחויבות) כמעט לא מכוסים.
8. **אפס AI חי בפרוד** ו־**אפס הכל ב-preview**.

## 4. תוכנית-המופת: הסימולטור

### 4.1 עיקרון-על: "שחקן + במאי + מאמן" (שלוש שכבות)
הסימולטור הנוכחי = שחקן בלבד. המופת מוסיף במאי ומאמן:

```
┌─ שחקן (Persona) ── מגיב בדמות, עקבי לטיפוס ולמצב הרגשי הנוכחי
├─ במאי (Director) ─ מנהל את הקשת: מצב-חיבור 0-100, ביטים ("עכשיו היא בודקת אותך"),
│                    מזריק אירועים (שקט מביך, הודעה בטלפון שלה, שאלה בוחנת)
└─ מאמן (Coach) ──── שקט בזמן-אמת (לחשן אופציונלי) + דיבריף עומק בסוף
```

**מימוש:** שדה `personaState` על `simulatorSessions`: `{connection: number, mood: string, beatIndex: number}`. כל תור: הבמאי מעדכן את המצב (חוקים דטרמיניסטיים + ניתוח-תור זול ב-Gemini), והפרומפט של הפרסונה מקבל את המצב העדכני. **ה-free-degradation נשמר:** בלי מפתח — הבמאי רץ על חוקים היוריסטיים בלבד (אורך תשובות, שאלות, מילות-רגש), והחוויה עדיין מדגימה קשת.

### 4.2 ספריית פרסונות מתורת אומנות-הקשר (הבידול האמיתי)
לא עוד "מיה נחמדה" — פרסונות שמגלמות את **הטיפולוגיה של אלעד** (טבלת סוגי האישיות + "מה תוקע אותי" + 3 רמות המשיכה משבוע 6 + ביטויים רגשיים):

**סכימת פרסונה מורחבת** (הרחבה אדיטיבית ל-`simulatorScenarios`):
```
personaArchetype: string        // הטיפוס מהטבלה של אלעד
attachmentStyle: string         // בטוח / חרד / נמנע (מהספר)
attractionProfile: string       // איזו רמת-משיכה דומיננטית אצלה: שכלית/רגשית/פיזית
triggers: string[]              // מה מכבה אותה ("תלונות", "חקירה", "התנשאות")
openers: string[]               // מה מקרב אותה ("סקרנות אמיתית", "פגיעות מדודה", "הומור עצמי")
beats: [{at, direction}]        // ביטים של הבמאי: "תור 3: היא זורקת מבחן-ביטחון"
phaseNumber: number             // איזה שלב-קורס התרחיש מאמן
```

**מטריצת כיסוי — מינימום 15 תרחישים (3 לכל שלב), נגזרים מתוכן השיעורים עצמם:**
| שלב | כישור | דוגמאות תרחישים חדשים |
|---|---|---|
| 1 גישה | גבולות + סיפור-פנימי | "היא מבקשת ממך לוותר על משהו שחשוב לך" · "דייט שמזלזל — תרגול גבול" |
| 2 תקשורת | רגש+צורך בלי האשמה | "שיחה אחרי אי-הבנה בוואטסאפ" · "היא משתפת קושי — הקשבה בלי לתקן" |
| 3 משיכה ומעבר | פתיחה, פלירט, דחייה | "פתיחת שיחה בבית קפה (קר!)" · "היא עונה במונוסילביות — להחיות או לשחרר" · "דחייה מנומסת — לצאת בכבוד" |
| 4 חיבור וכימיה | פגיעות, עומק | "דייט 3 — לעבור משיחת-חולין לעומק" · "היא שואלת על העבר שלך" |
| 5 מחויבות | DTR, ערכים | "שיחת 'לאן זה הולך' (יוזם/מגיב)" · "אי-הסכמה ראשונה כזוג" |

כל תרחיש חדש נכתב **מתוך תוכן השיעור** (יש 27K מילים!) בקול-אלעד, ועובר את צינור הסקילים (hebrew-writer→elad-voice) לפני seed. **Preview-one-first:** תרחיש אחד מלא לאישור סגנון → ואז השאר.

### 4.3 דיבריף-מאמן עומק (המשוב הוא המוצר)
אחרי כל סימולציה, במקום ציון+רשימות:
1. **מד-חיבור לאורך השיחה** — גרף קטן (SVG, יש דפוס בפרויקט) של ה-connection per turn — "פה עלית, פה איבדת אותה".
2. **2-3 רגעי-מפתח מצוטטים:** "אמרת: '...' → מה קרה: היא נסגרה (חיבור 62→41) → מה היה עובד: ... (מתוך שיעור 6.2: שלוש רמות המשיכה)" — **עם דיפ-לינק לשיעור.** זה סוגר את לולאת קורס↔תרגול בשני הכיוונים.
3. **רדאר כישורים (5 צירים = 5 שלבי הקורס):** יוזמה · הבעת-רגש · אומץ · פגיעות/עומק · הובלה. נשמר per-session → מאפשר גרף התקדמות.
4. **תרגיל אחד לפעם הבאה** ("בדייט הבא של הסימולטור, נסה לשאול שאלה רגשית אחת לפחות").
5. **משחק חוזר (Replay):** כפתור "נסה שוב את הרגע הזה" — ממשיך את השיחה מרגע-המפתח עם אותה פרסונה באותו מצב. (fork של ההודעות עד אותו turn — מוטציה פשוטה.)

מימוש: הרחבת `analyzeConversation` לסכימת JSON עשירה (moments[], radar{}, drill) + fallback היוריסטי מורחב. הכל דרך `generateChat` הקיים.

### 4.4 לחשן (מאמן בזמן-אמת, opt-in)
טוגל "🎧 מאמן מלווה": אחרי כל תגובת-פרסונה מופיעה שורת-לחישה חד-פעמית ("היא רמזה על משהו חשוב — שאל עליו"). ברירת-מחדל כבוי (לא לשבור אימרסיה). חינם: תבנית לפי-שלב; עם מפתח: Gemini עם prompt קצרצר. 

### 4.5 מדדי התקדמות ו"מוכנות לדייט"
- `simulatorSkillsHistory` נגזר מהדיבריפים → עמוד `/simulator/progress`: רדאר משתנה בזמן, streak תרגולים, "ציון מוכנות" משוקלל.
- שילוב בגיימיפיקציה הקיימת (XP, badges — המערכת כבר בנויה!): תג "שרדת דייט קשה", "5 תרגולים בשבוע".
- הצגה בדשבורד + ב-LessonAdvisor (הרחבת `getLessonPracticeStats` הקיים).

### 4.6 קול (אופק ב') 
פרסונה מדברת: Gemini TTS (חינם, מוכח ב-voice-chat-claude) על תגובות הפרסונה + כפתור השתקה. בהמשך: STT לתשובות המשתמש = תרגול דיבור אמיתי. מאחורי feature-flag. **זה ה-wow שממיר בשיווק — אבל רק אחרי שהליבה מושלמת.**

## 5. תוכנית-המופת: היועץ החכם

### 5.1 RAG — "היועץ שקרא הכל" (בדפוס זהות המוכח)
**קורפוס:** 75 שיעורים (27K מילים) + הספר (87K מילים) + 12 סיכומי-שבוע + טיפולוגיה. ~120K מילים ≈ 400-600 צ'אנקים של 250-400 מילים.

**ארכיטקטורה (Convex-native):**
1. `scripts/build-knowledge-index.mjs` (העתקת build-book-index.mjs של זהות): קריאת המקורות → צ'אנקים עם מטא (`source: lesson|book|typology`, `lessonId?`, `weekNumber?`, `title`) → embedding ב-gemini-embedding-001 (768d, חינם).
2. טבלת `knowledgeChunks` ב-Convex עם **`vectorIndex`** (יכולת מובנית של Convex — אין תלות חיצונית): `{text, source, title, lessonId?, weekNumber?, phaseNumber?, embedding: v.array(v.float64())}`.
3. ב-`advisor.ask`: embed לשאלה → `ctx.vectorSearch` top-5 (עם פילטר-שלב רך: boost לצ'אנקים מהשלב הנוכחי) → בלוק-עיגון בפרומפט: "קטעי מקור מתוך הקורס והספר — הסתמך עליהם וציין את שם השיעור".
4. **Free-degradation נשמר:** אין מפתח → אין embedding → התבניות הקיימות. יש מפתח → יועץ שמצטט את אלעד מילה-במילה ומקשר לשיעור.
5. **בונוס מיידי:** אותו אינדקס משרת גם את הדיבריף של הסימולטור (סעיף 4.3.2 — "מתוך שיעור X") וגם את הצ'אט הכללי `/chat`.

### 5.2 זיכרון-לומד (persistent learner profile)
טבלת `advisorProfiles`: `{userId, goals[], strugglesSummary, winsSummary, lastTopics[], updatedAt}` — תקציר מתגלגל שמתעדכן (זול, פעם בשיחה) מ: שיחות יועץ + תוצאות סימולציות + התקדמות + ציוני בחנים. הפרומפט של היועץ מקבל: פרופיל + סטטיסטיקות תרגול + שלב נוכחי. תוצאה: "בפעם שעברה סיפרת שקשה לך עם פתיחת שיחה — ראיתי שתרגלת פעמיים בסימולטור והשתפרת מ-45 ל-72. בוא נעלה מדרגה."
+ סגירת backlog קיים: persist תורות ה-LessonAdvisor ל-`chatSessions` (היום stateless).

### 5.3 יועץ פרואקטיבי (לא רק מגיב)
- **פתיח מותאם** בכניסה לשיעור: במקום פתיח-שלב גנרי — פתיח מחושב מהפרופיל ("סיימת את שבוע 5 — לפני שממשיכים, השיעור הזה מתחבר בדיוק לקושי שסיפרת עליו").
- **הצעת-תרגול חכמה:** `getRecommendedScenario` משודרג לשקלל גם את רדאר-הכישורים (הכישור החלש ביותר = התרחיש המומלץ), לא רק את השלב.
- **צ'ק-אין שבועי** (cron קיים בפרויקט — `crons.ts`): התראת "3 ימים בלי תרגול" דרך מערכת ההתראות הקיימת.
