יש לבצע את השלבים לפי הסדר. כל שלב תלוי בקודם.

שלב 1: Convex Production

1

יצירת Convex production deployment

הריצו את הפקודה הזו כדי ליצור deployment חדש ל-production:

npx convex deploy

הפקודה תשאל אם ליצור deployment חדש. בחרו Yes ותנו שם (למשל: haderech-production).

⚠️ שמרו את ה-Deployment URL שתקבלו — תצטרכו אותו בשלב הבא
2

Seed data - טעינת תוכן ראשוני

אחרי ה-deploy, טענו את הקורס ואת הנתונים:

# טעינת הקורס + שיעורים + PDFs
npx convex run seedHaderech:seedHaderechCourse

# טעינת תוכן יומי (ציטוטים, טיפים, אתגרים)
npx convex run dailyContent:seedDailyContent

# טעינת בלוג פוסטים
npx convex run blog:seedBlogPosts
✓ אחרי הרצה — בדקו ב-Convex Dashboard שהנתונים נטענו

שלב 2: Clerk Production

3

יצירת Clerk Production Instance

היכנסו ל-dashboard.clerk.com:

  • לחצו על ה-instance הנוכחי → Production tab
  • צרו Production instance חדש
  • העתיקו את NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY (מתחיל ב-pk_live_)
  • העתיקו את CLERK_SECRET_KEY (מתחיל ב-sk_live_)
⚠️ מפתחות production מתחילים ב-pk_live_ ו-sk_live_ (לא pk_test_)
4

הגדרת Clerk Webhook

ב-Clerk Dashboard → Webhooks:

  • צרו webhook endpoint חדש: https://YOUR-DOMAIN.co.il/api/webhooks/clerk
  • בחרו events: user.created, user.updated, user.deleted
  • העתיקו את CLERK_WEBHOOK_SECRET

שלב 3: Vercel Deploy

5

חיבור לפרויקט Vercel

אם לא חיברתם עדיין:

npx vercel                    # קישור ראשוני
npx vercel env pull .env.local  # משיכת env vars

או דרך vercel.com/new — יבוא מ-GitHub.

6

הגדרת Environment Variables ב-Vercel

ב-Vercel Dashboard → Settings → Environment Variables, הוסיפו (ראו טבלת משתנים):

# חובה
NEXT_PUBLIC_CONVEX_URL=https://YOUR-PROD.convex.cloud
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_live_...
CLERK_SECRET_KEY=sk_live_...
NEXT_PUBLIC_APP_URL=https://haderech.co.il

# רצוי
CLERK_WEBHOOK_SECRET=whsec_...
NEXT_PUBLIC_GA_MEASUREMENT_ID=G-XXXXXXX
NEXT_PUBLIC_VAPID_PUBLIC_KEY=(כבר מוגדר בקוד)
⚠️ Scope: הגדירו לכל הסביבות (Production, Preview, Development)
7

Deploy ל-Production

npx vercel --prod

או פשוט git push — Vercel ירוץ אוטומטית אם מחובר ל-GitHub.

✓ אחרי deploy — בדקו ש-/api/health מחזיר status: "ok"

שלב 4: דומיין

8

חיבור דומיין (אופציונלי)

ב-Vercel Dashboard → Settings → Domains:

  • הוסיפו דומיין: haderech.co.il (או הדומיין שלכם)
  • עדכנו DNS records לפי ההוראות של Vercel
  • SSL certificate ניתן אוטומטית
  • עדכנו NEXT_PUBLIC_APP_URL ב-Vercel env vars

שלב 5: Stripe (אופציונלי - כשמוכנים לתשלומים)

9

הגדרת Stripe

היכנסו ל-dashboard.stripe.com:

  • העבירו ל-live mode (לא test)
  • צרו מוצרים + מחירים (Basic, Premium)
  • העתיקו את STRIPE_SECRET_KEY ו-NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY
  • צרו Webhook endpoint: https://YOUR-DOMAIN/api/webhooks/stripe
  • Events: checkout.session.completed, customer.subscription.updated, customer.subscription.deleted
⚠️ Stripe לא חובה לשלב הראשון — האתר עובד גם בלי

שלב 6: Resend (אופציונלי - מיילים)

10

הגדרת Resend למיילים

היכנסו ל-resend.com:

  • צרו API key
  • אמתו דומיין (DNS records)
  • הגדירו RESEND_API_KEY ב-Vercel
